# xClaw vs Hermes Agent 对比分析与车机功能增强方案

## 一、架构对比

| 维度 | xClaw (当前) | Hermes Agent | 差距 |
|------|-------------|--------------|------|
| **记忆系统** | 内存 dict，50条消息滑动窗口，无持久化 | SQLite + FTS5 全文搜索，MEMORY.md/USER.md 跨会话持久化 | 🔴 严重缺失 |
| **技能学习** | 无 | 完成复杂任务后自动提炼为可复用技能文件 | 🔴 严重缺失 |
| **自我进化** | 无 | GEPA 引擎，百次评估收敛 | 🟡 可后期实现 |
| **定时任务** | 无 | Cron 调度器，支持周期性任务 | 🔴 车机核心需求 |
| **上下文压缩** | 无，仅截断 | LLM 摘要压缩旧对话 | 🟡 重要 |
| **工具系统** | 18个工具，大部分无参数 | 70+ 工具，28个工具集，自动注册 | 🔴 工具不完整 |
| **流式响应** | 定义了接口但未使用 | 全链路流式 | 🟡 体验提升 |
| **结构化日志** | 无（structlog 在依赖中但未使用） | 完整可观测性 | 🟡 运维需求 |
| **安全护栏** | 仅确认提示 | 指令审批、速率限制、审计日志、沙箱 | 🔴 车机安全关键 |
| **多步规划** | 单次工具调用 | 迭代规划、自纠正、多轮工具调用 | 🔴 复杂场景需求 |
| **平台接入** | 4个平台 | 20+ 平台 | 🟢 已够用 |
| **模型支持** | 9个适配器 | 200+ 模型 | 🟢 已够用 |

## 二、车机场景专属缺失

| 功能 | 说明 | 优先级 |
|------|------|--------|
| **座椅加热/方向盘加热** | Vehicle 类已实现但未暴露为工具 | P0 |
| **限速模式** | Vehicle 类已实现但未暴露为工具 | P1 |
| **代客模式** | Vehicle 类已实现但未暴露为工具 | P1 |
| **Homelink 车库门** | Vehicle 类已实现但未暴露为工具 | P2 |
| **软件更新调度** | Vehicle 类已实现但未暴露为工具 | P2 |
| **位置获取/导航** | 有位置数据但无工具暴露 | P0 |
| **地理围栏** | .env.example 有配置但无实现 | P1 |
| **定时空调/充电** | 无实现 | P0 |
| **充电历史/能耗分析** | 无实现 | P1 |
| **车辆健康诊断** | 无实现 | P1 |
| **多车支持** | 仅支持单车 | P2 |

## 三、实施计划

### Phase 1: 核心基础设施（本次实施）
1. ✅ 持久化记忆系统（SQLite + FTS5）
2. ✅ 结构化日志（structlog）
3. ✅ 安全护栏（速率限制、审计日志）
4. ✅ 补全工具系统（座椅加热、限速、位置等）
5. ✅ 定时任务调度器
6. ✅ 上下文压缩
7. ✅ 多步规划引擎
8. ✅ 技能自动学习

### Phase 2: 车机专属功能（后续）
1. 地理围栏触发器
2. 充电历史与能耗分析
3. 车辆健康诊断报告
4. 多车管理
5. 流式响应
